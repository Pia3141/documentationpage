Introduction

CSS stands for 'Cascadian Style Sheets'. Similar to HTML, CSS is not a programming, but rather a stylesheet language. CSS describes how an HTML webpage is displayed.

Syntax

CSS takes elements of the HTML page and assigns attributes, such as colors, borders and even animations. The syntax used is: 

    selector { 
      attribute1: value1;
      attribute2: value2;}

There can be more than one attribute assigned to an element. The attributes and their values are seperated with a colon (:). Several attributes are seperated by a semicolon (;) inside braces ({}).

Integration

How do we tell the Browser what stylesheet to use on a specific HTML document ?

  • Inline CSS

    We can add CSS in our HTML document itself, using the style tag in the head of our HTML file.
    Example:

    <style> h1 { color: blue; } </style>

    Note: This is not a common practice, as it clutters your HTML document and is overal rather impractical.

  • External CSS

    We can add the styles in a sepperate document and link it to our HTML documents in the head. This allows not only for an organized collection of styles, but also enables you to link one CSS file to several HTML files to keep a consistant style between pages.

    <link rel="stylesheet" href="styles.css"/ >
Selectors

In the example above you can see that tags can be used to select elements from the HTML document. These are not restricted to the visible tags of text-declarations (such as h1, p, em, ...), but also include structural tags such as main, nav, body and even the html tag itself.

What if you only want to change the style of one or several objects of a type? This is where the class and ids you assigned to elements in your HTML file come in handy.

  • You can target an element with its id, by putting a # infront of it.
  • You can target classes by adding a . in front of their name.
  • To select every element on the page, you can use the * selector.

Example:

          
  * {
    font-size: 20px;
    }

  h1 {
    font-weight: 30;
    }

  .class {
    background-color: blue;
    }

  #id {
    color: red;
    }
Comments

Sometimes, you will need to describe what you are doing so that others who read the code understand what a certain part is for.
You also might need to temporarily remove a line of code from your CSS file.
In both cases, consider using CSS comments! Lines that are declared as a comment are ignored by the browser and thus not displayed. Every programming language has its own way a comment is declared. 

          /* This is a 
          css comment 
          over several lines */
